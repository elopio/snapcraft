snapcraft.internal.lifecycle.execute(step, project_options, part_names=None)[source]¶

Execute until step in the lifecycle for part_names or all parts.

Lifecycle execution will happen for each step iterating over all the available parts, if part_names is specified, only those parts will run.

If one of the parts to execute has an after keyword, execution is forced until the stage step for such part. If part_names was provided and after is not in this set, an exception will be raised.

Parameters:
  • step (str) – A valid step in the lifecycle: pull, build, prime or snap.
  • project_options – Runtime options for the project.
  • part_names (list) – A list of parts to execute the lifecycle on.
Raises:

RuntimeError – If a prerequesite of the part needs to be staged and such part is not in the list of parts to iterate over.
Returns:

A dict with the snap name, version, type and architectures.

snapcraft.internal.parser module¶

snapcraft-parser

Parse a wiki index of parts and output formatted YAML for use by ‘snapcraft’. ‘snapcraft’ uses the YAML to find non-local parts referred to by a project’s ‘snapcraft.yaml’ file. See ‘snapcraft update’ and ‘snapcraft define’ for more details.

Usage:
snapcraft-parser [options]
Options:

-h –help show this help message and exit -v –version show program version and exit -d –debug print debug information while executing

(including backtraces)
-i –index=<filepath> a file containing a part index.
(‘http://wiki.ubuntu.com/snapcraft/parts?action=raw‘)
-o –output=<filename> where to write the parsed parts list.
(‘snap-parts.yaml’)
